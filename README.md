## 2. 系統總覽與定位

### 2.1 系統定位（摘自原明文件並對齊現況）
- 系統名稱：`憨吉聯合大醫院系統`
- 技術：`HTML` + `CSS` + `JavaScript`
- 部署：直接於瀏覽器開啟 `web/index.html`（或自 `index.html` 進入系統入口）

- 執行型態：多頁式網頁（MPA）
- 主入口：`web/index.html`

- 系統管理：主檔維護、公告管理、個資流程監管  

### 2.4 功能模組總覽

- 查詢／退掛／轉掛  
- 排班管理、規則排班／給號設定

## 3. 資料庫與檔案結構

### 3.1 格式說明

#### 3.1.1 `OPDREGROOMTBL`（掛號排班檔）

**檔案卡**
- 主鍵：`P(chDate, chPNC, chRoom)`
- 關聯：`chSecNo -> 科別檔`、`chDocNo -> 醫師檔`
- 主要用途：排班、停診／代診、給號序列、限額控制

**欄位格式（摘要）**

| Key | 欄位英文 | 欄位中文 | 型態 | 長度 | 格式／範例 | 說明 |
|---|---|---|---|---|---|---|
| P | `chDate` | 日期 | `char` | 7 | `0861225` | 看診日 |
| P | `chPNC` | 時段 | `char` | 1 | `1/2/3` | 早／午／晚 |
| P | `chRoom` | 診間代號 | `char` | 4 | `0107` | 診間鍵值 |
|  | `chSecNo` | 科別代號 | `char` | 2 | `22` | 關聯科別檔 |
|  | `chDocNo` | 醫師代號 | `char` | 5 | `0105` | 關聯醫師檔 |
|  | `chIfStop` | 是否停診 | `char` | 1 | `0/1` | 1=停診 |
|  | `chIfSubst` | 是否代診 | `char` | 1 | `0/1` | 1=代診 |
|  | `chSubDocNo` | 代診醫師代號 | `char` | 5 | `0106` | 代診醫師 |
|  | `chSubRoom` | 代診診間 | `char` | 4 | `0102` | 合併掛號診間 |
|  | `chNoMethod` | 給號方式 | `char` | 1 | `1` | 對照給號方式檔 |
|  | `intBeginNo` | 起始號 | `int` | - | `1` | 預設起號 |

### 4.6 公告資料

### 4.7 系統事件紀錄（Event Log）

## 5. 系統分頁與功能說明

### 5.1 病人系統

#### 5.1.1 首頁 `patient-system.html`
- 病人流程入口（掛號、查詢／退掛／轉掛、病患總覽、公告）。

#### 5.1.2 掛號 `registration.html`
- 支援初診／複診掛號、院區選擇、科別／醫師連動  
- 初診必填健保卡號  
- 自動給病歷號與看診號  
- 列印正式掛號單  

#### 5.1.3 查詢／退掛／轉掛 `lookup.html`
- 依姓名／身分證／病歷號／日期查詢  
- 退掛需記錄原因  
- 轉掛建立新掛號並保留原掛號關聯  

#### 5.1.4 病患總覽 `patients.html`
- 顯示病患主檔（病歷號、身分證、健保卡號、基本資料）  
- 最近掛號追蹤  

#### 5.1.5 公告管理 `announcements.html`
- 新增／刪除公告  
- 病人端查閱最新公告  

### 5.2 批價櫃台

#### 5.2.1 首頁 `billing-system.html`
- 收費與報表入口  

#### 5.2.2 批價收費 `billing.html`
- 依掛號記錄試算費用  
- 計算項目：掛號費、診察費、部分負擔、醫囑金額  
- 依身份別／折扣別計算總額  
- 列印正式收據  

#### 5.2.3 報表 `reports.html`
- 指標總覽（掛號、病患、醫囑、收費）  
- 科別掛號量  
- 醫師掛號量與收入  

#### 5.2.4 櫃台查詢
- 使用 `lookup.html` 執行查詢、退掛、轉掛  

### 5.3 醫生系統

#### 5.3.1 首頁 `doctor-system.html`
- 科別、排班、規則、醫囑入口  

#### 5.3.2 科別管理 `departments.html`
- 新增科別、新增醫師並指定科別  
- 刪除限制：科別底下有醫師不可刪；已有掛號紀錄醫師不可刪  

#### 5.3.3 排班管理 `schedule.html`
- 新增／刪除醫師時段排班  

#### 5.3.4 規則與給號 `rules.html`
- 規則排班（星期、時段、限掛）  
- 給號設定（起始號、限額、給號方式）  

#### 5.3.5 醫囑 `orders.html`
- 醫囑類型：藥品、處置、治療、檢查、醫材、其他  
- 總量公式：`劑量 × 頻率 × 天數`  

### 5.4 管理系統

#### 5.4.1 首頁 `management-system.html`
- 管理維護入口（科別、排班、規則、公告、資料管理）  

#### 5.4.2 管理維護
- 帳務與掛號流程一致性檢查  
- 院區、科別、排班、公告維護  

### 5.5 資料管理

#### 5.5.1 頁面 `data-management.html`
- 個資驗證：`身分證號 + 健保卡號`  
- 驗證後可修改：姓名、健保卡號、性別、生日、電話、地址等  

#### 5.5.2 規則
- 未驗證不得修改  
- 初診建檔需先有健保卡號  

### 5.6 補充頁面
- `index.html`：系統入口  
- `presentation.html`：PPT 互動簡報

## 6. 核心業務流程、刪除規則與列印

> 本章收錄 `docs/憨吉聯合大醫院系統明文件.md` 第 4 節。

### 6.1 掛號流程
1. 先輸入身分證查詢是否既有病患。  
2. 若無資料，建立新病患並自動給病歷號（`001` 起）。  
3. 選擇日期、科別、時段後，系統依排班過濾醫師。  
4. 系統自動試算該醫師該時段該日下一號。  
5. 送出後建立掛號紀錄並寫入事件紀錄。  

### 6.2 退掛／轉掛流程
- 退掛：將掛號狀態改為 `cancelled`，保存退掛原因。  
- 轉掛：原掛號標記 `transferred`，產生新掛號並串接來源／目標 ID。  

### 6.3 醫囑流程
- 醫囑總量計算：`總量 = 劑量 × 頻率 × 天數`  
- 小計：`總量 × 單價`  
- 每筆醫囑與掛號關聯，方便後續批價與統計。  

### 6.4 批價流程
1. 選擇掛號紀錄。  
2. 帶入掛號費／診察費／部份負擔。  
3. 彙總該掛號醫囑金額。  
4. 依身份別計算自付金額，再依折扣別套用折扣。  
5. 確認後產生批價紀錄。  

### 6.5 刪除規則（管理安全）
- 科別刪除：若底下仍有醫師，不可刪除。  
- 醫師刪除：若已有掛號歷史，不可刪除。  
- 刪除醫師時，會同步移除其排班與規則排班資料。  

### 6.6 正式列印樣板
- 掛號系統可列印「門診掛號單」：含病患、科別、醫師、時段、看診號碼。  
- 批價系統可列印「門診收據」：含收據號、費用明細、折扣與總額。  
- 列印採獨立視窗樣板，不會直接印整個操作頁面。

## 9. 系統規格

> 本章對齊 `docs/系統規格.md`，與前文重複處以「規格條列」統一表述。

### 9.1 功能規格

#### 9.1.1 掛號
- 初診判斷：以身分證查無病患即為初診。  
- 初診必填：`健保卡號`。  
- 院區必填：中央／介仁／建國院區。  
- 給號邏輯：依醫師、時段、日期計算下一號。  

#### 9.1.2 查詢／退掛／轉掛
- 查詢條件：姓名、身分證、病歷號、日期。  
- 退掛需記錄原因。  
- 轉掛會保留原掛號關聯資訊。  

#### 9.1.3 批價收費
- 試算基礎：掛號費、診察費、部分負擔、醫囑費用。  
- 身份別：健保／自費。  
- 折扣別：一般／員工／弱勢。  
- 收費確認後產生收據記錄，支援列印。  

#### 9.1.4 醫囑
- 醫囑類型：藥品、處置、治療、檢查、醫材、其他。  
- 總量公式：`劑量 × 頻率 × 天數`。  
- 小計：`總量 × 單價`。  

#### 9.1.5 管理維護
- 科別、醫師、排班、規則、公告可維護。  
- 刪除限制：科別有醫師不可刪；醫師已有掛號紀錄不可刪。  

#### 9.1.6 個資驗證修改
- 驗證條件：身分證號 + 健保卡號。  
- 驗證成功後開放修改個資欄位。  
- 修改後即時回寫病患主檔。  

### 9.2 資料規格（摘要鍵值）
| 項目 | 說明 |
|---|---|
| 病患顯示主鍵 | `medicalRecordNo`（病歷號） |
| 病患內部鍵 | `id`（系統） |
| 身份識別 | `nationalId` |
| 個資驗證組合 | `nationalId` + `healthCardNo` |
| 掛號業務唯一性（概念） | `doctorId` + `timeSlotId` + `regDate` + `regNumber` |

### 9.3 非功能規格
- 前端技術：`HTML` / `CSS` / `JavaScript`。  
- 資料儲存：瀏覽器 `localStorage`。  
- 互動相容：桌面瀏覽器優先。  
- 匯入匯出：**不以主導覽提供全域 JSON 匯入／匯出操作入口**；個資變更集中於 `data-management.html`（與產品現況一致；若日後於管理後台恢復匯備份，應另立權限與稽核）。  

### 9.4 安全與限制
- 本版本為教學／展示型系統，無後端帳號權限控管。  
- 個資驗證僅為前端流程驗證，不等同正式資安機制。  
- 正式上線環境建議補強：帳號權限、後端 API 驗證、稽核軌跡與不可否認記錄、資料庫加密與傳輸保護。

## 11. 驗收與維運建議

### 11.1 驗收檢核（摘自 `docs/系統規格.md` 第 8 節）
- 初診未填健保卡號不可掛號。  
- 身分證 + 健保卡號驗證錯誤不可進入個資修改。  
- 批價試算結果與收據列印數字一致。  
- 退掛／轉掛後查詢結果狀態正確。  
- 醫師／科別刪除限制符合規則。  

### 11.2 維運建議
- 定期備份 `localStorage` 資料（可擴充匯出備份機制）。  
- 針對交易流程補上操作紀錄追蹤。  
- 規劃後端化（API + DB）時，先依第 3 章與第 4 章建立 ERD 與遷移策略。
