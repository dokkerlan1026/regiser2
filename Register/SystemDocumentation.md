# 憨吉聯合大醫院 - 掛號系統擴充說明文件

為滿足不同管理流程及確保掛號資料的嚴謹度，系統擴充了數個新的資料表以及獨立的管理頁面。以下為系統新增功能與預設配備的說明介紹。

## 資料庫與資料表設計 (Database & Tables)
本系統採用本地輕量級關聯式資料庫 `SQLite` (`clinic.db`) 進行資料儲存。主要架構如下：

1. **Departments (科別表)**
   - 包含系統所有的醫學科別，目前預設為：內科、外科、皮膚科、眼科、牙科、小兒科、婦產科。
2. **TimeSlots (時段表)**
   - 掛號能夠選擇的時段，預設提供三段時間：上午、下午、晚上。
3. **Doctors (醫師表)**
   - 記錄診所擁有的醫師名單，每一位醫師均有其專屬的科別（Foreign Key 至 Departments）。
   - **本次更新**：每個科別均各自配屬「兩名」專科醫師，共計 14 位醫師，以便在掛號時能有效分流。
4. **DoctorSchedules (醫師排班表)**
   - 負責串接醫師與時段的關聯。當病患在主畫面選擇了「科別」與「時段」後，系統會自動在該資料表中撈取符合條件的醫師供選。
5. **Patients (病患資料表)**
   - 儲存曾經掛號過的病患相關資訊。
   - **本次核心更新**：將「病歷號 (MedicalRecordNo)」與「身分證/居留證字號 (NationalId)」欄位分離，使欄位職責更為明確。
6. **Registrations (掛號紀錄表)**
   - 整合掛號歷史，關聯了病患 (PatientId)、醫師排班 (TimeSlotId、DoctorId) 與掛號日期，提供後續追蹤。

## 新增管理頁面說明 (New UI Forms)

除了原有的 **`Reg.cs` (主掛號系統)** 將病歷與身分證各自透過專屬文字框及新增的下拉式選單隔開外，系統亦在主畫面右上方新增了功能選擇，點擊即可存取下列兩張獨立的表單。

### 1. 醫師排班表 (`DoctorScheduleForm.cs`)
- **功能介紹**：供醫院後台管理人員檢視診所內醫師與時段的綁定概況。
- **介面細節**：透過一個自動綁定的 `DataGridView` 以唯讀的形式列舉所有診間與排班資料（包含科別名稱、時段、醫師姓名）。此畫面可將龐大的班表資訊一目了然地匯成報表呈現給行政人員。

### 2. 病患總覽管理表 (`PatientManagementForm.cs`)
- **功能介紹**：記錄所有看診病患的核心個資以及其對應的掛號資料。
- **介面細節**：使用關聯式的 SQL 查詢 (`LEFT JOIN`)，系統會自動將掛號狀態連動進入表單清單。這意味著管理人員可以藉由此頁面檢視「誰在哪天看了哪位醫師的門診」，以及該病人的性別、生日等。生日部分現已受到限制，必須填寫如 `09201015`（即8碼純數字）的精確格式，減少資料建置錯誤。

### 3. 科別與醫師管理頁面 (`DepartmentManagementForm.cs`)
- **功能介紹**：作為院內的行政參考，列出所有科別清單與其轄下負責的醫師。
- **介面細節**：透過 SQLite `GROUP_CONCAT`，能將多位專科醫師彙整於同一個資料行，便於閱覽。這構成了系統的獨立第 4 個表單頁面。

### 4. 掛號統計報表頁面 (`ReportForm.cs`)
- **功能介紹**：展示每位醫師及科別目前的總掛號量，提供運營參考。
- **介面細節**：利用 `GROUP BY` 及聚合函數 `COUNT()` 對關聯的 `Registrations` 紀錄計算業績與接診量，是整個系統的第 5 個表單頁面。
