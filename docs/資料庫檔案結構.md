# 憨吉聯合大醫院 - 資料庫檔案結構

> 依據 `2.掛號系統檔案結構.doc` 彙整，並對應現行網頁版資料模型。  
> 本版已調整為「規格文件格式」：分群清單 + 檔案卡 + 欄位格式表。

## 1. 格式說明

- 鍵值符號：`P` = 主鍵（Primary Key）、`S` = 索引鍵（Index Key）
- 欄位型態以原始文件為準：`char` / `int` / `tinyint` / `real` / `varchar`
- 日期欄位多採民國格式字串（如 `0861225`）

## 3. 核心交易檔格式

### 3.1 `OPDREGROOMTBL`（掛號排班檔）

**檔案卡**

- 主鍵：`P(chDate, chPNC, chRoom)`
- 關聯：`chSecNo -> 科別檔`、`chDocNo -> 醫師檔`
- 主要用途：排班、停診/代診、給號序列、限額控制

**欄位格式（摘要）**


| Key | 欄位英文            | 欄位中文   | 型態     | 長度  | 格式/範例       | 說明      |
| --- | --------------- | ------ | ------ | --- | ----------- | ------- |
| P   | `chDate`        | 日期     | `char` | 7   | `0861225`   | 看診日     |
| P   | `chPNC`         | 時段     | `char` | 1   | `1/2/3`     | 早/午/晚   |
| P   | `chRoom`        | 診間代號   | `char` | 4   | `0107`      | 診間鍵值    |
|     | `chSecNo`       | 科別代號   | `char` | 2   | `22`        | 關聯科別檔   |
|     | `chDocNo`       | 醫師代號   | `char` | 5   | `0105`      | 關聯醫師檔   |
|     | `chIfStop`      | 是否停診   | `char` | 1   | `0/1`       | 1=停診    |
|     | `chIfSubst`     | 是否代診   | `char` | 1   | `0/1`       | 1=代診    |
|     | `chSubDocNo`    | 代診醫師代號 | `char` | 5   | `0106`      | 代診醫師    |
|     | `chSubRoom`     | 代診診間   | `char` | 4   | `0102`      | 合併掛號診間  |
|     | `chNoMethod`    | 給號方式   | `char` | 1   | `1`         | 對照給號方式檔 |
|     | `intBeginNo`    | 起始號    | `int`  | -   | `1`         | 預設起號    |
|     | `intCurNextNo`  | 現場目前號  | `int`  | -   | `12`        | 現場下一給號  |
|     | `intPreNextNo`  | 預約目前號  | `int`  | -   | `22`        | 預約下一給號  |
|     | `chNoList`      | 掛號給號序列 | `char` | 999 | `001101...` | 已給號標記   |
|     | `intPreLimit`   | 預約限額   | `int`  | -   | `12/999`    | 999=不限  |
|     | `intTotalLimit` | 限掛人數   | `int`  | -   | `99/999`    | 999=不限  |


### 3.2 `OPDREGULARROOMTBL`（規則排班檔）

**檔案卡**

- 主鍵：`P(chWeekDay, chPNC, chRoom)`
- 主要用途：建立週期規則排班，供每日排班複製

**欄位格式（摘要）**


| Key | 欄位英文            | 欄位中文 | 型態     | 長度  | 格式/範例   | 說明       |
| --- | --------------- | ---- | ------ | --- | ------- | -------- |
| P   | `chWeekDay`     | 星期   | `char` | 1   | `0~6`   | 0=日, 1=一 |
| P   | `chPNC`         | 時段   | `char` | 1   | `1/2/3` | 早/午/晚    |
| P   | `chRoom`        | 診間代號 | `char` | 4   | `0107`  | 診間鍵值     |
|     | `chSecNo`       | 科別代號 | `char` | 2   | `22`    | 關聯科別檔    |
|     | `chDocNo`       | 醫師代號 | `char` | 5   | `0105`  | 關聯醫師檔    |
|     | `intPreLimit`   | 預約限額 | `int`  | -   | `999`   | 預設不限     |
|     | `intTotalLimit` | 限掛人數 | `int`  | -   | `999`   | 預設不限     |
|     | `chNoMethod`    | 給號方式 | `char` | 1   | `1`     | 對照給號方式   |
|     | `intBeginNo`    | 起始號  | `int`  | -   | `1`     | 預設起號     |


### 3.3 `OPDREGDIAGNOSISTBL`（掛號看診檔）

**檔案卡**

- 主鍵：`P(chDate, chPNC, chRoom, intRegNo)`
- 索引建議：`S(chMRNo)`、`S(chRegMethod)`、`S(chDocNo)`
- 主要用途：記錄掛號、看診、退掛、轉掛、批價狀態

**欄位格式（摘要）**


| Key | 欄位英文             | 欄位中文   | 型態     | 長度  | 格式/範例         | 說明             |
| --- | ---------------- | ------ | ------ | --- | ------------- | -------------- |
| P   | `chDate`         | 日期     | `char` | 7   | `0861225`     | 看診日            |
| P   | `chPNC`          | 時段     | `char` | 1   | `1/2/3`       | 早/午/晚          |
| P   | `chRoom`         | 診間代號   | `char` | 4   | `0107`        | 診間鍵值           |
| P   | `intRegNo`       | 序號     | `int`  | -   | `5`           | 就醫序號           |
| S   | `chMRNo`         | 病歷號    | `char` | 10  | `K212245698`  | 病患識別           |
|     | `chRegFM`        | 初複診掛號  | `char` | 1   | `0/1/2`       | 初診未報到/初診已報到/複診 |
|     | `chRegMethod`    | 掛號方式   | `char` | 2   | `01~16/33/77` | 現場/電話/語音/電腦代掛  |
|     | `chInsuSeqNo`    | 健保卡序號  | `char` | 3   | `A1`          | 批價系統使用         |
|     | `chInsuType`     | 身分別    | `char` | 3   | `11`          | 健保/自費分類        |
|     | `chDCTType`      | 折扣別    | `char` | 3   | `112`         | 折扣分類           |
|     | `chIfWithdraw`   | 退掛否    | `char` | 1   | `0/1`         | 1=退掛           |
|     | `chWithdrawDate` | 退掛日期   | `char` | 7   | `0861225`     | 沖銷基準           |
|     | `chWithdrawTime` | 退掛時間   | `char` | 5   | `15:30`       | 退掛時間           |
|     | `chIfRegSTC`     | 轉掛狀態   | `char` | 1   | `0/1/2`       | 退掛/轉掛/不計服務量    |
|     | `intReceiptNo`   | 掛號費收據號 | `int`  | -   | `123456`      | 收據對應           |

## 4. 現行網頁版資料結構格式（localStorage）

### 4.1 根物件結構


| 欄位                | 型態              | 說明           |
| ----------------- | --------------- | ------------ |
| `departments`     | `Array<Object>` | 科別清單         |
| `doctors`         | `Array<Object>` | 醫師清單         |
| `timeSlots`       | `Array<Object>` | 時段清單         |
| `cities`          | `Array<Object>` | 縣市/行政區       |
| `campuses`        | `Array<Object>` | 院區（中央/介仁/建國） |
| `doctorSchedules` | `Array<Object>` | 排班資料         |
| `scheduleRules`   | `Array<Object>` | 規則排班         |
| `patients`        | `Array<Object>` | 病患主檔（含健保卡號）  |
| `registrations`   | `Array<Object>` | 掛號交易         |
| `orders`          | `Array<Object>` | 醫囑交易         |
| `billings`        | `Array<Object>` | 批價收費         |
| `announcements`   | `Array<Object>` | 公告資料         |
| `events`          | `Array<Object>` | 事件紀錄         |


### 4.2 主要物件格式（摘要）


| 物件                | 必要欄位                                                                          | 說明   |
| ----------------- | ----------------------------------------------------------------------------- | ---- |
| `patients[]`      | `id`, `medicalRecordNo`, `name`, `nationalId`, `healthCardNo`                 | 病患主檔 |
| `registrations[]` | `id`, `patientId`, `doctorId`, `regDate`, `timeSlotId`, `regNumber`, `status` | 掛號交易 |
| `billings[]`      | `id`, `registrationId`, `items`, `totalAmount`, `paidAt`                      | 收費交易 |
| `orders[]`        | `id`, `registrationId`, `orderType`, `dose`, `frequency`, `days`              | 醫囑資料 |

## 5. 對映關係（Legacy -> Web）


| Legacy 欄位                   | Web 欄位                     | 說明      |
| --------------------------- | -------------------------- | ------- |
| `OPDREGDIAGNOSISTBL.chMRNo` | `patients.medicalRecordNo` | 病歷號對映   |
| `chInsuSeqNo`               | `patients.healthCardNo`    | 健保卡資訊   |
| `chIfWithdraw/chIfRegSTC`   | `registrations.status`     | 退掛/轉掛狀態 |
| `intRegNo`                  | `registrations.regNumber`  | 掛號序號    |


---

## 6. 備註

- Legacy 結構屬大型醫院主機系統，欄位完整且偏交易導向。
- 現行網頁版為教學/流程展示模型，欄位較精簡。
- 後續若接正式資料庫，建議先由第 4 章物件格式建立 ERD，再逐步擴充第 3 章欄位。
