## 1. 文件目的與範圍

### 1.1 文件目的
- 定義本系統之功能範圍、資料結構、流程規則與驗收基準。

- 病人系統
- 批價櫃台

- 管理系統
- 資料管理（個資驗證修改）

## 2. 系統總覽

### 2.1 技術架構
- 前端技術：HTML / CSS / JavaScript
- 儲存方式：瀏覽器 `localStorage`
- 執行型態：多頁式網頁（MPA）
- 主入口：`web/index.html`

### 2.2 角色與權責
- 掛號人員：掛號、查詢、退掛、轉掛、病患資料查核
- 批價櫃台：費用試算、收費確認、收據列印、報表查詢
- 醫師/醫療行政：醫囑開立、排班設定、規則設定
- 系統管理：主檔維護、公告管理、個資流程監管

## 3. 資料庫檔案結構

### 3.1 Legacy HIS 主要檔案

#### 3.1.1 基本主檔
- `DIVISIONTBL`：科別主檔
- `DOCTORTBL`：醫師主檔
- `COUNTYTBL`：縣市主檔
- `TOWNTBL`：鄉鎮主檔

#### 3.1.2 掛號與排班
- `OPDREGROOMTBL`：掛號排班檔（每日）
- `OPDREGULARROOMTBL`：規則排班檔（週規則）
- `OPDREGDIAGNOSISTBL`：掛號看診檔（交易主檔）
- `OPDGIVENUMBERTBL`：給號方式對照檔
- `OPDFLOWANONYMTBL`：流水號無名氏檔

#### 3.1.3 科別/時段/診別
- `GENSECTIONTBL`：院內科別檔
- `GENDECSECTIONTBL`：申報科別檔
- `GENTIMEDIVISIONTBL`：時段檔
- `OPDROOMTYPETBL`：診別檔
- `OPDSPECIALROOMTBL`：特殊診檔

#### 3.1.4 收費與診斷書
- `GENHOSPITALTBL`：轉診醫院對照檔
- `OPDREGFEETBL`：掛號費檔
- `OPDDIAGNOSISBOOKTBL`：診斷書檔
- `OPDBOOKMANAGETBL`：診斷書管理檔

#### 3.1.5 統計檔
- `OpdRegPeopleTbl`：掛號人數統計
- `OpdStcDoctorTbl`：醫師服務統計
- `OpdStcRegWayTbl`：掛號方式統計
- `OpdStcRegServiceTbl`：掛號員服務統計

### 3.2 關鍵欄位摘要

#### 3.2.1 `OPDREGROOMTBL`（掛號排班檔）
- 主鍵：`chDate`, `chPNC`, `chRoom`
- 控制：`chIfStop`, `chIfSubst`, `chSubDocNo`, `chSubRoom`
- 給號：`chNoMethod`, `intBeginNo`, `intCurNextNo`, `intPreNextNo`, `chNoList`
- 限額：`intPreLimit`, `intTotalLimit`

#### 3.2.2 `OPDREGULARROOMTBL`（規則排班檔）
- 主鍵：`chWeekDay`, `chPNC`, `chRoom`
- 規則：`intPreLimit`, `intTotalLimit`, `chNoMethod`, `intBeginNo`

#### 3.2.3 `OPDREGDIAGNOSISTBL`（掛號看診檔）
- 主鍵：`chDate`, `chPNC`, `chRoom`, `intRegNo`
- 核心：`chMRNo`, `chRegFM`, `chRegMethod`, `chDocNo`
- 收費：`chInsuType`, `chDCTType`, `chInsuSeqNo`, `intReceiptNo`
- 異動：`chIfWithdraw`, `chWithdrawDate`, `chWithdrawTime`, `chIfRegSTC`

### 3.3 現行網頁版資料模型（localStorage）
- `departments`：科別主檔
- `doctors`：醫師主檔
- `timeSlots`：時段主檔
- `cities`：縣市/行政區主檔
- `campuses`：院區主檔（中央/介仁/建國）
- `doctorSchedules`：醫師排班
- `scheduleRules`：規則排班
- `patients`：病患主檔（含身分證、健保卡號）
- `registrations`：掛號交易檔
- `orders`：醫囑檔
- `billings`：批價收費檔
- `announcements`：公告檔
- `events`：系統事件紀錄

### 3.4 Legacy 與 Web 對映
- `OPDREGDIAGNOSISTBL.chMRNo` -> `patients.medicalRecordNo`
- `chInsuSeqNo` -> `patients.healthCardNo` + `billings.insurance`
- 退掛/轉掛欄位 -> `registrations.status/cancelReason/transferToRegistrationId`
- 給號欄位 -> `config + registrations.regNumber`

## 4. 系統分頁與功能說明

### 4.1 病人系統

#### 4.1.1 首頁 `patient-system.html`
- 病人流程入口（掛號、查詢/退掛/轉掛、病患總覽、公告）。

#### 4.1.2 掛號 `registration.html`
- 支援初診/複診掛號、院區選擇、科別/醫師連動
- 初診必填健保卡號
- 自動給病歷號與看診號
- 列印正式掛號單

#### 4.1.3 查詢/退掛/轉掛 `lookup.html`
- 依姓名/身分證/病歷號/日期查詢
- 退掛需記錄原因
- 轉掛建立新掛號並保留原掛號關聯

#### 4.1.4 病患總覽 `patients.html`
- 顯示病患主檔（病歷號、身分證、健保卡號、基本資料）
- 最近掛號追蹤

#### 4.1.5 公告管理 `announcements.html`
- 新增/刪除公告
- 病人端查閱最新公告

### 4.2 批價櫃台

#### 4.2.1 首頁 `billing-system.html`
- 收費與報表入口

#### 4.2.2 批價收費 `billing.html`
- 依掛號記錄試算費用
- 計算項目：掛號費、診察費、部分負擔、醫囑金額
- 依身份別/折扣別計算總額
- 列印正式收據

#### 4.2.3 報表 `reports.html`
- 指標總覽（掛號、病患、醫囑、收費）
- 科別掛號量
- 醫師掛號量與收入

#### 4.2.4 櫃台查詢
- 使用 `lookup.html` 執行查詢、退掛、轉掛

### 4.3 醫生系統

#### 4.3.1 首頁 `doctor-system.html`
- 科別、排班、規則、醫囑入口

#### 4.3.2 科別管理 `departments.html`
- 新增科別
- 新增醫師並指定科別
- 刪除限制：
  - 科別底下有醫師不可刪
  - 已有掛號紀錄醫師不可刪

#### 4.3.3 排班管理 `schedule.html`
- 新增/刪除醫師時段排班

#### 4.3.4 規則與給號 `rules.html`
- 規則排班（星期、時段、限掛）
- 給號設定（起始號、限額、給號方式）

#### 4.3.5 醫囑 `orders.html`
- 醫囑類型：藥品、處置、治療、檢查、醫材、其他
- 總量公式：`劑量 × 頻率 × 天數`

### 4.4 管理系統

#### 4.4.1 首頁 `management-system.html`
- 管理維護入口（科別、排班、規則、公告、資料管理）

#### 4.4.2 管理維護
- 帳務與掛號流程一致性檢查
- 院區、科別、排班、公告維護

### 4.5 資料管理

#### 4.5.1 頁面 `data-management.html`
- 個資驗證：`身分證號 + 健保卡號`
- 驗證後可修改：姓名、健保卡號、性別、生日、電話、地址等

#### 4.5.2 規則
- 未驗證不得修改
- 初診建檔需先有健保卡號

### 4.6 補充頁面
- `index.html`：系統入口
- `presentation.html`：PPT 互動簡報

## 5. 系統規格

### 5.1 掛號規格
- 初診判斷：身分證查無病患即初診
- 初診必填健保卡號
- 院區必填：中央/介仁/建國院區
- 給號：依醫師 + 時段 + 日期計算下一號

### 5.2 查詢/退掛/轉掛規格
- 查詢條件：姓名、身分證、病歷號、日期
- 退掛須記錄原因
- 轉掛須保留原掛號關聯

### 5.3 批價收費規格
- 試算項目：掛號費、診察費、部分負擔、醫囑費用
- 身份別：健保/自費
- 折扣別：一般/員工/弱勢
- 收費完成產生收據紀錄並可列印

### 5.4 醫囑規格
- 類型：藥品、處置、治療、檢查、醫材、其他
- 總量：`劑量 × 頻率 × 天數`
- 小計：`總量 × 單價`

### 5.5 管理維護規格
- 可維護：科別、醫師、排班、規則、公告
- 刪除限制：
  - 科別有醫師不可刪
  - 醫師有掛號紀錄不可刪

### 5.6 個資驗證修改規格
- 驗證條件：身分證號 + 健保卡號
- 驗證成功後開放修改個資欄位
- 儲存後即時回寫病患主檔

### 5.7 非功能規格
- 前端技術：HTML/CSS/JavaScript
- 資料儲存：`localStorage`
- 相容性：桌面瀏覽器優先

### 5.8 安全與限制
- 本版為教學/展示型，無完整後端權限控管
- 個資驗證為前端流程驗證，不等同正式資安
- 正式上線建議補強：
  - 帳號權限
  - 後端 API 驗證
  - 稽核軌跡
  - 資料加密與傳輸保護

## 6. 驗收與維運建議

### 6.1 驗收檢核
- 初診未填健保卡號不可掛號
- 身分證 + 健保卡號驗證錯誤不可進入個資修改
- 批價試算與收據列印金額一致
- 退掛/轉掛後查詢狀態正確
- 醫師/科別刪除限制符合規則

### 6.2 維運建議
- 定期備份 `localStorage` 資料（可擴充匯出備份機制）
- 針對交易流程補上操作紀錄追蹤
- 規劃後端化（API + DB）時，先依第 3 章建立 ERD 與遷移策略
