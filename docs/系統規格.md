# 憨吉聯合大醫院 - 系統規格

## 1. 文件目的
- 定義現行網頁版系統之功能範圍、流程規則、資料與驗證規格。
- 作為後續維運、擴充與驗收基準。

## 2. 系統範圍
- 病人系統
- 批價櫃台
- 醫生系統
- 管理系統
- 資料管理（個資驗證修改）

## 3. 角色與權責
- 掛號人員：掛號、查詢、退掛、轉掛、病患查核。
- 批價櫃台：費用試算、收費確認、收據列印、報表查詢。
- 醫師/醫療行政：醫囑開立、排班設定、規則設定。
- 系統管理：主檔維護、公告管理、個資流程監管。

## 4. 功能規格

### 4.1 掛號
- 初診判斷：以身分證查無病患即為初診。
- 初診必填：`健保卡號`。
- 院區必填：中央/介仁/建國院區。
- 給號邏輯：依醫師+時段+日期計算下一號。

### 4.2 查詢/退掛/轉掛
- 查詢條件：姓名、身分證、病歷號、日期。
- 退掛需記錄原因。
- 轉掛會保留原掛號關聯資訊。

### 4.3 批價收費
- 試算基礎：掛號費、診察費、部分負擔、醫囑費用。
- 身份別：健保/自費。
- 折扣別：一般/員工/弱勢。
- 收費確認後產生收據記錄，支援列印。

### 4.4 醫囑
- 醫囑類型：藥品、處置、治療、檢查、醫材、其他。
- 總量公式：`劑量 × 頻率 × 天數`。
- 小計：`總量 × 單價`。

### 4.5 管理維護
- 科別、醫師、排班、規則、公告可維護。
- 刪除限制：
  - 科別有醫師不可刪。
  - 醫師已有掛號紀錄不可刪。

### 4.6 個資驗證修改
- 驗證條件：身分證號 + 健保卡號。
- 驗證成功後開放修改個資欄位。
- 修改後即時回寫病患主檔。

## 5. 資料規格（摘要）
- 病患主鍵：`medicalRecordNo`（顯示）與系統 `id`（內部）。
- 身份識別：`nationalId`。
- 個資驗證：`nationalId + healthCardNo`。
- 掛號交易鍵：`doctorId + timeSlotId + regDate + regNumber`（業務唯一）。

## 6. 非功能規格
- 前端技術：HTML/CSS/JavaScript。
- 資料儲存：瀏覽器 `localStorage`。
- 互動相容：桌面瀏覽器優先。
- 匯入匯出：目前改為個資管理頁，不提供全域 JSON 匯入匯出操作入口。

## 7. 安全與限制
- 本版本為教學/展示型系統，無後端帳號權限控管。
- 個資驗證僅為前端流程驗證，不等同正式資安機制。
- 上線正式環境建議加上：
  - 帳號權限
  - 後端 API 驗證
  - 稽核軌跡與不可否認記錄
  - 資料庫加密與傳輸保護

## 8. 驗收檢核建議
- 初診未填健保卡號不可掛號。
- 身分證+健保卡號驗證錯誤不可進入個資修改。
- 批價試算結果與收據列印數字一致。
- 退掛/轉掛後查詢結果狀態正確。
- 醫師/科別刪除限制符合規則。
